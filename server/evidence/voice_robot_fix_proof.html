Live OAuth + MoveIt proof

Voice robot fix: the UR10 now observes, targets, executes, and verifies.

This page documents the failed-run diagnosis, code fix, and live proof captured through the real OpenAI Codex OAuth backend against the local MoveIt MCP server.

Root cause

The agent prompt and safety layer referenced stale tools like moveit_get_robot_status. The live MCP server exposes moveit_get_current_pose and canonical MoveIt workflow tools. The bridge also duplicated legacy aliases.

Fix

Aligned tool exposure, validation, prompt text, context updates, and relative-motion repair with the live canonical MCP schema.

Result

2/2
Live robot movement turns passed with execution verification.

What changed

1
Bridge. RobotMCPBridge now de-duplicates legacy aliases and prefers canonical moveit_* tools.
2
Safety. Local validation now accepts the live schemas: target_pose, waypoints, current-pose observation, and plan-and-execute workflows.
3
Agent. The processor refreshes current pose before Codex turns, stores it as context, and repairs missing relative target_pose values for simple commands like up/down.
4
Timeout. MCP client-session timeout increased to 30s so real MoveIt plan/execute calls can finish.
