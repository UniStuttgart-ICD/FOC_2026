The failure is contractual.

moveit_plan_pick(auto) can look executable after solving only prepositioning. The agent then believes a pick is ready, even though grasp, attach, lift, and verification still depend on later work.

Current slippery state

Preposition success leaks as pick success. The LLM has to improvise a multi-stage robot workflow from partial planner evidence.

Required correction

A pick result is executable only when the returned executable corresponds to the declared pick workflow. Partial results stay diagnostic.

Target tool shape.

Plan a task solution, then execute that exact solution. MTC can sit behind the boundary later, but the agent should never author raw MTC stages.

Research tips pulled in.

The plan now incorporates the useful guidance from the agentic MoveIt workflow findings page instead of treating it as background reading.

Stage evidence

Task results carry stage reports, failed stage, candidate attempts, solver, cost, clearance, blocker, and suggested next tool.

Scene binding

Planning results echo scene snapshot id, planning frame, selected object, pose age, and object-context source.

Approval payload

Execution is bound to one solution id, object, expected movement, scene snapshot, user turn, and timestamp.

Object proof

Pick success waits for attachment verification; place success waits for release or placed-object evidence.

Replay first

Replay artifacts record tool order, typed outputs, policy, validation, approval, execution, verification, and terminal job events.

Tool tiers

The surface stays narrow: observation, planning, execution, diagnostic, and admin. No broad ROS mutation tools.

Parallelization review.

The original plan had good technical content but overlapping write sets. The revised plan separates by repo, file ownership, and dependency wave so a controller can dispatch safely.

Rule one

No two workers edit the same file in the same wave.

Rule two

Each worker gets owned files, forbidden files, and a verification command.

Rule three

Review each package for spec compliance before code quality.
