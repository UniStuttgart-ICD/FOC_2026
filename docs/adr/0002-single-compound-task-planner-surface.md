# Single Compound Task Planner Surface

Agent Orchestration exposes one model-visible task-level manipulation planner, `moveit_plan_compound_task`, for MTC-backed goals such as `hold`, `release`, `move_and_release`, and `pick_place`. `moveit_plan_pick_task` and `moveit_plan_place_task` may remain MCP/manual internals during migration, but they are hidden from the Pipecat model-visible tool surface so the agent does not choose between competing task-planning contracts. This keeps preview, approval, execution-contract, and release semantics in one place at the cost of making the compound planner responsible for ordinary manipulation workflows.
