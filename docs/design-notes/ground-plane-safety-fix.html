Recovery Flow

The actual code path that turns a failed MoveIt plan into an agent retry opportunity.

Detect failed planner result

robot_control/job_worker.py

The worker parses the returned tool JSON. A structured ok: false, failed verification, or failed execution marks the queued job FAILED instead of completed.

error = _tool_result_error(result) fail(job, error, result=result)

Keep the evidence

robot_control/job_board.py

Failed jobs now keep the raw planner payload. Queued jobs also carry the original user text so the recovery prompt knows the intent, not just the tool name.

RobotJob.user_text RobotJob.result FAILED event {"error", "result"}

Ask the agent again

agent_control/langchain_agent_processor.py

The failed job notification builds a recovery turn for the graph agent. It includes the user request, tool arguments, error summary, and full planner result.

_failed_job_recovery_prompt(...) graph.run_turn(recovery_turn)

Repair retry-only speech

agent_control/langgraph_robot_agent.py

If the recovery answer says I can try or will retry without a tool call, the graph treats it as an unfinished action and asks for exactly one MoveIt action tool.

FUTURE_PROMISE_TERMS += "i can try" repair_missing_action

Queue the real retry

agent_control/robot_job_submission.py

The retry only counts when a MoveIt action tool is submitted as a new robot job. The worker then runs that job and reports completion or failure from MoveIt.

submit_tool(..., user_text=...) Queued robot job ...
What did not change: there is no hard-coded z=0 preflight block. MoveIt still decides feasibility; the agent now gets enough data to respond and retry honestly.

Code Changes

Planner failure is no longer flattened into a generic notification.

Planner failure packet

RobotJobWorker still lets MoveIt try the requested motion. If MoveIt returns ok: false, the job is failed and the raw planner result is kept on the job event.

error = tool_result_error(result) if error:   fail(job, error, result=result)

Agent recovery turn

LangChainAgentProcessor now turns a failed queued job into a recovery prompt containing the original user request, tool arguments, error summary, and planner result.

Original user request: touch z equals zero Tool arguments: {"z": 0.0, ...} Planner result: {"ok": false, ...}

Retry claim repair

LangGraphRobotAgent treats recovery text such as "I can try..." as a future action claim. It must repair that into an actual MoveIt action tool call before speaking the retry.

if text says "I can try" and no tool call:   require one MoveIt action tool   queue retry before the spoken response
