Design Note · MoveIt Pick Workflow

Cartesian Pick Follow-up Fix

The pick workflow now preserves MoveIt's staged follow-up instruction and plans the local Cartesian pick after a verified preposition execution succeeds.

Date: 2026-05-15 Scope: Agent 1 issue only Status: Fixed and tested

Observed Failure

Around the 13:30 local run, the robot received a pick request for dynamic_5. MoveIt planned the free-motion preposition above the object, and that was the only motion that reached execution.

The local Cartesian pick segment was present in the MoveIt result, but the agent did not request the follow-up Cartesian planning call after the preposition finished.

Root Cause

moveit_plan_pick returned raw.next_action.after_success with a follow-up moveit_plan_pick call using planning_strategy="cartesian".

RobotContextStore only remembered the executable preposition plan_name. The follow-up tool and arguments were discarded, so the graph had no instruction to continue into the Cartesian planner.

Fixed Flow

1. Plan Pick MoveIt returns a staged preposition plan plus after_success.
2. Store Context The pending plan keeps after_success_tool and after_success_arguments.
3. Execute Preposition Verified execution must pass before the follow-up is considered.
4. Plan Cartesian Pick The stored moveit_plan_* follow-up is called automatically.
5. Await Execution The Cartesian pick plan becomes pending; execution still requires explicit user intent.

Implementation

  • Context preservation: server/robot_control/context.py stores after_success_tool and after_success_arguments on PendingExecutablePlan.
  • Result parsing: _after_success_action extracts valid raw.next_action.after_success tool metadata from successful planning output.
  • Graph continuation: server/agent_control/langgraph_robot_agent.py calls _execute_after_success_action after verified moveit_execute_plan success.
  • Safety boundary: The automatic continuation is restricted to moveit_plan_* tools. It does not silently execute the Cartesian pick motion or gripper steps.

Regression Coverage

  • Context test: test_robot_context_preserves_pick_follow_up_after_success proves the follow-up survives context storage.
  • Graph test: test_verified_preposition_execute_continues_with_after_success_pick_plan proves a successful preposition execution calls the Cartesian pick planner and records its returned plan.
  • Queue behavior: Existing worker-backed planning tests remain green, including the one-plan-per-turn queue guard.
